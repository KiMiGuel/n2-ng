# N2-ng Design Specification

**Date:** 2026-07-15  
**Status:** Approved for implementation  
**Project location:** `/home/kali/n2-ng/`  
**Target platform:** Kali Linux USB boot on MacBook (Alfa AWUS036ACHM, TP-Link Archer T2U Nano, NetGear Atheros adapters)

## 1. Overview

N2-ng is a single-file Python/tkinter GUI tool that wraps the aircrack-ng suite (`airmon-ng`, `airodump-ng`, `aireplay-ng`) into a unified, lightweight capture interface. It is capture-focused: it does **not** perform cracking.

## 2. Goals

- One-window interface — no multi-terminal juggling.
- Auto monitor-mode detection and start/stop with cleanup on exit or crash.
- Live airodump-ng CSV parsing into a sortable, color-coded network table.
- Target lock: click a network → lock channel + BSSID, open capture panel, show clients.
- Intuitive aireplay-ng attack buttons with full-word labels and a 3-second cancelable countdown.
- Auto-save captures to `~/hs/n2-ng/<ESSID>_<BSSID>/` with timestamps.
- Real-time WPA handshake / PMKID detection with green banner + modal dialog.
- Auto-convert captures to `.22000` via `hcxpcapngtool` when available.
- Root handling via startup sudo relaunch with a secure password dialog.
- Dependency check at startup with real `apt` install commands / GitHub links.
- Dark hacker aesthetic, large buttons, responsive layout suitable for a MacBook screen.

## 3. Non-goals

- No cracking inside the tool (no `aircrack-ng` dictionary attacks).
- No manual channel hopping once a target is locked.
- No confusing single-letter flags in the UI.
- No matplotlib or heavy GUI frameworks; tkinter only.

## 4. Deliverables

- `n2-ng.py` — single main application file.
- `requirements.txt` — minimal; tkinter is standard library.
- `install.sh` — dependency checker / installer helper.
- `README.md` — exact run commands and adapter notes.

## 5. Architecture

`n2-ng.py` is organized into small, focused classes:

| Class | Responsibility |
|-------|----------------|
| `AirmonManager` | Detect adapters, start/stop monitor mode, restore state on exit. |
| `AirodumpWorker` | Run `airodump-ng`, parse CSV, emit network/client updates via queue. |
| `AttackController` | Build and run `aireplay-ng` commands with countdown/cancel. |
| `CaptureManager` | Create capture dirs, detect handshake/PMKID, run `hcxpcapngtool`, merge/fix captures. |
| `WpsScanner` | Run `wash` / `reaver --scan` and parse WPS-enabled targets. |
| `SignalGraph` | tkinter Canvas mini-graph of locked target PWR. |
| `DeauthHistory` | Log deauth events and prevent rapid repeats. |
| `N2NgApp` | tkinter UI assembly and main loop. |

All subprocess I/O runs in daemon threads. Threads push `(event_type, payload)` tuples into a `queue.Queue`. The tkinter main thread drains the queue via `after(150, poll_queue)`.

## 6. Root handling & dependency check

1. App starts as normal user.
2. If not root, show a secure tkinter password dialog.
3. Re-execute the script with `sudo -S`, piping the password into sudo's stdin.
4. Once root, run dependency checks for:
   - `aircrack-ng` (`airmon-ng`, `airodump-ng`, `aireplay-ng`)
   - `hcxpcapngtool` from `hcxtools` (optional)
   - `wash` from `reaver` (optional)
   - `mergecap` from `wireshark-common` (optional, for merging caps)
   - `pcapfix` (optional, for repairing truncated caps)
5. If a tool is missing, prompt the user with the real install command:
   - `sudo apt update && sudo apt install -y aircrack-ng`
   - `sudo apt install -y hcxtools`
   - `sudo apt install -y reaver`
   - `sudo apt install -y wireshark-common`
   - `sudo apt install -y pcapfix`
   Or GitHub links for manual build:
   - https://github.com/aircrack-ng/aircrack-ng
   - https://github.com/ZerBea/hcxtools
   - https://github.com/t6x/reaver-wps-fork-t6x

## 7. GUI layout

Single resizable tkinter window, dark theme (black/gray/green), default ~1200×700, minimum ~1000×600.

- **Top toolbar:** adapter dropdown, band dropdown (2.4GHz / 5GHz / Both), "Start Monitor", "Stop Monitor", "WPS Scan", big channel lock indicator pill.
- **Left panel:** sortable `Treeview` of networks with columns PWR, Beacons, #Data, CH, MB, ENC, CIPHER, AUTH, ESSID, BSSID.
  - Row colors: Green=OPN, Red=WEP, Yellow=WPA, White=WPA2, Cyan=WPA3.
  - Hidden ESSIDs display as `[Hidden]` until revealed via probe or deauth.
  - Right-click menu: "Copy BSSID", "Copy ESSID".
- **Right panel:**
  - Target info card (ESSID, BSSID, channel, signal).
  - Clients sub-table.
  - Signal graph canvas.
  - Live capture file size monitor.
  - Attack buttons (front-and-center): "Deauthenticate All Clients", "Deauthenticate Specific Client".
  - Collapsible "Legacy WEP Attacks" section: "Fake Authentication", "ARP Replay", "Chopchop", "Fragmentation".
  - "Capture Handshake" auto-deauth toggle with interval selector (10s / 30s / 60s).
  - Capture history list with right-click export / merge / fix.
- **Bottom status bar:** adapter/monitor interface, scan state, last handshake/PMKID, root indicator.

**Channel lock indicator pill:**
- Red: "SCANNING ALL" when hopping all channels.
- Green: "LOCKED: CH X" when a target is locked.

## 8. Scanning & target lock

**Scanning mode:**
- Band dropdown maps to `--band` flag:
  - 2.4GHz → `--band bg`
  - 5GHz → `--band a`
  - Both → `--band abg`
- Command: `airodump-ng --write-interval 1 -w /tmp/n2ng_scan --output-format csv --band <band> <mon_iface>`
- A daemon thread reads `/tmp/n2ng_scan-01.csv` every ~1.5 s.
- Parser splits CSV on empty line into AP list and client list.
- Networks stored in dict keyed by BSSID; treeview refreshed only on CSV change.
- Hidden ESSIDs (empty or `<length: 0>` ESSID field) display as `[Hidden]` and use folder name `hidden_<BSSID>`.
- When a client's Probed ESSIDs or airodump-ng ESSID field reveals the real name, update the row and target card in real-time.

**Target lock:**
- Double-click a row or click "Lock Target".
- Run `iw dev <mon_iface> set channel <CH>` to prevent channel mismatch.
- Restart airodump-ng with `-w <base_dir>/<timestamp>`; airodump-ng will append `-01.cap`, `-01.csv`, etc.:
  `airodump-ng -c <CH> --bssid <BSSID> -w ~/hs/n2-ng/<ESSID>_<BSSID>/<timestamp> --output-format csv <mon_iface>`
- Channel lock indicator pill turns green and shows "LOCKED: CH X".
- Right panel updates with target info, clients, signal graph, and live capture file size monitor.
- Capture file size is polled every 1 s and displayed in human-readable form (e.g., 12.4 MB).

**Multi-adapter support:**
- Dropdown lists `wlan*` / `wlp*` interfaces from `airmon-ng` and `ip link`.
- Switching adapters cleans up old monitor mode and starts fresh on the new one.

## 9. Attack panel

**Primary attack buttons (front-and-center, large):**

| Button | Command |
|--------|---------|
| Deauthenticate All Clients | `aireplay-ng -0 10 -a <BSSID> <mon_iface>` |
| Deauthenticate Specific Client | `aireplay-ng -0 10 -a <BSSID> -c <CLIENT_MAC> <mon_iface>` (requires a selected client row) |

**Legacy WEP Attacks (collapsible section, hidden by default):**

| Button | Command |
|--------|---------|
| Fake Authentication | `aireplay-ng -1 0 -a <BSSID> -h <OUR_MAC> <mon_iface>` |
| ARP Replay | `aireplay-ng -3 -b <BSSID> -h <OUR_MAC> <mon_iface>` |
| Chopchop | `aireplay-ng -4 -b <BSSID> -h <OUR_MAC> <mon_iface>` |
| Fragmentation | `aireplay-ng -5 -b <BSSID> -h <OUR_MAC> <mon_iface>` |

- 3-second countdown with "Cancel" and "Execute Now" for every attack button.
- Attack runs in a daemon thread; stdout/stderr streamed to the internal log pane.
- Attack buttons disabled while an attack is running.

**Deauth history:**
- Log every deauth with BSSID, client (or "ALL"), timestamp, ESSID.
- Warn if repeating the same client within 60 seconds.

## 10. Capture manager

**Auto-save:**
- Base dir: `~/hs/n2-ng/<ESSID>_<BSSID>/`
- ESSID is sanitized for directory names:
  - Hidden ESSIDs → `hidden_<BSSID>`
  - Strip characters: `/ \ : * ? " < > |`
  - Replace spaces with `_`
  - Truncate to 50 characters max
- Filename: `capture_YYYY-MM-DD_HH-MM-SS`
- Full example: `/home/kali/hs/n2-ng/MyWiFi_AA-BB-CC-DD-EE-FF/capture_2026-07-15_12-30-00-01.cap`

**Detection (poll-based, no stdout parsing):**
- Every 5 s, poll the active `.cap` file with `hcxpcapngtool -o <temp>.22000 <cap>`.
- Fallback if `hcxpcapngtool` is missing: `aircrack-ng <cap> -J <temp>`.
- Only flash the green banner / show the modal when the conversion yields a non-empty valid hash file.
- Same logic for PMKID (check `.22000` output for PMKID entries).
- Avoid relying on airodump-ng `WPA handshake:` stdout text because it often does not pipe correctly through subprocess.

**Notifications:**
- Green status banner: "WPA Handshake Captured" / "PMKID Captured".
- Modal dialog: ESSID, BSSID, capture path; buttons "Open Folder", "OK".

**Conversion:**
- If `hcxpcapngtool` is installed:
  `hcxpcapngtool -o <capture>.22000 <capture>.cap`
- `.22000` written beside `.cap`.

**Merge & fix:**
- Capture history panel groups captures by ESSID/BSSID.
- "Merge Selected" uses `mergecap -w <output>.cap <inputs...>` if available.
- "Fix" uses `pcapfix <input.cap> <output.cap>` for truncated captures.
- If tools are missing, show real install command.

**Quick export:**
- Right-click capture → "Copy hashcat command" (`hashcat -m 22000 <file> <wordlist>`) or "Copy .22000 to clipboard".

## 11. Bonus features

- **WPS Scan:** `wash -i <mon_iface>` (fallback `reaver -i <mon_iface> --scan`); show BSSID, ESSID, WPS version, locked/unlocked.
- **Signal graph:** tkinter Canvas line graph of last 60 PWR samples.
- **Auto-deauth loop:** "Capture Handshake" mode sends `aireplay-ng -0 5 -a <BSSID>` at a user-selected interval (10s / 30s / 60s) until handshake detected, then stops.
- **Multi-adapter support:** described in §8.
- **Quick export:** described in §10.
- **Client deauth history:** described in §9.

## 12. Error handling & cleanup

- `atexit` + `WM_DELETE_WINDOW` handler stops all subprocesses and restores managed interface state.
- Remove temp scan files `/tmp/n2ng_*`.
- Subprocess failures logged in status bar/log pane, not blocking popups.
- Unexpected `airodump-ng` death triggers one restart; second failure shows "Scan failed — check adapter".
- Adapter disappearance pauses scan and refreshes dropdown.
- Deauth buttons disabled until target locked and monitor mode active.

## 13. Dependencies

Required:
- Python 3.x (tkinter included)
- aircrack-ng
- root privileges

Optional:
- hcxtools (`hcxpcapngtool`)
- reaver (`wash`)
- wireshark-common (`mergecap`)
- pcapfix

## 14. Testing approach

- Syntax check: `python3 -m py_compile n2-ng.py`
- Dependency check: run `./install.sh`
- Run as normal user and verify sudo relaunch dialog.
- With supported adapter, verify monitor mode start/stop, scanning on 2.4GHz / 5GHz / Both, target lock, channel-set via `iw dev`, channel lock indicator pill, deauth countdown, handshake detection banner (via `.cap` polling), and `.22000` conversion.
- Verify hidden ESSID display as `[Hidden]`, folder naming `hidden_<BSSID>`, and real-time update when ESSID is revealed.
- Verify capture file size monitor updates live.
- Verify right-click "Copy BSSID" / "Copy ESSID".
- Verify auto-deauth interval selector (10s / 30s / 60s).
- Verify Legacy WEP Attacks section is collapsible.
- Test cleanup: close window and verify `mon*` interface is removed and temp files deleted.
